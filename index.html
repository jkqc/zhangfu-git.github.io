javascript "RegExp类型"

javascript的RegExp类型支持正则表达式，我们经常会用正则对提交的数据进行验证。创建一个正则表达式：

var expression = / pattern/ flags;

其实pattern 是正则表达式，flags是一个或多个标志。我们进行会看到：

/正则表达式/g    //g表示全局模式：匹配所有的项，直到没有匹配项停止
/正则表达式/m    //表示多行模式
/正则表达式/i    //表示不区分大小写

RegExp有几个比较常用的实例方法

  • exec() //为捕获组而设计的，返回第一个匹配项信息的数组
  • test() //返回true 或者false，看目标字符串与某个模式是否匹配,基本这个是最常用的一个实例方法

具体用法看下面例子：

var text = ‘This is zhangfu, hi zhangfu’;
var pattern = /zhangfu/;
pattern.exec(text) //[zhangfu]
pattern.test(text); //true

在用nwe RegExp(),构造函数创建正在表达式每次都会新建一个RegExp实例，而字面量创建不会(/zhangfu/g),还有就是在用new RegExp()传入字面量表达式的时候要对某些字符进行转义处理

/\[bc\]at/  ====>  "\\[bc\\]at"

《javascript高级程序设计》笔记05

Date类型

js创建一个日期对象很简单 new Date();Date没有传入日期的毫秒数者获取当前的日期。

Date.parse()

Date.parse接收一个表示日期的字符串，然后返回相对应的日期毫秒数,通过它我们就可以创建指定的日期对象：

var myDate = new Date(Date.parse("Aug 13, 2016"));
//平时我们创建指定日期最多的方式应该是直接传入日期字符串
var myDate = new Date("Aug 13, 2016");  //其实后台还是调用了Date.parse()
日期格式化
  • toDateString() //显示星期几、月、日和年 “Sat Aug 13 2016”
  • toTimeString() //显时、分、秒和时区 “10:10:02 GMT+0800 (CST)”
  • toLocaleDateString() //本地星期几、月、日、年
  • toLocaleTimeString() //本地时、分、秒
注意: 日期对象在不同的浏览器中实现有许多奇怪的行为，其中有一种倾向是将超出范围的值替换为当前值。例如： January 32, 2007在有的浏览器是解析为’February 1, 2007’而Opera倾向于插入当前月份的当前日期 January 21, 2007(都是21日)

日期对象还有很多的组件方法，例如：

getTime()           //返回日期毫秒值
getFullYear()       //4位数的年份
getMonth()         //返回月份
getDate()          //返回月份的天数
getDay()           //返回星期几（0是星期日）
getHours()        //返回小时数
getMinutes()      //返回分钟数
getSeconds()      //返回秒数

当然还有很多其他组件方法，以上是比较常用的

在项目中我们可能会用moment.js这个库来操作日期和时候,用它我们可以很方便的获取我们想要的格式的日期和时间。

浏览器样式兼容问题

在写pc端页面的时候，我们经常要考虑一些浏览器的兼容问题（特别是万恶的IE），特别是针对IE的一些hack写法：

p { _color:red }   *html P { color:red }     //IE6 专用
p { +color: red; *color:red }           //IE6, 7 专用
p { *+color:red }                      //IE7 专用
/*\\*//*/p{ +color:red }/**/            //IE8 专用
:root p{ color:red\9 }                 //IE9 专用
有时候发现这些hack写法不好使，我们就可以巧用条件注释语句（）来实现针对不同浏览器写不同的样式来解决这个问题。
<!--[if IE 6]><html class="ie lt-ie8"><![endif]-->
<!--[if IE 7]><html class="ie lt-ie8"><![endif]-->
<!--[if IE 8]><html class="ie ie8"><![endif]-->
<!--[if IE 9]><html class="ie ie9"><![endif]-->
<!--[if !IE]><!--> <html> <!--<![endif]-->

这是简书中使用的一段条件注释语句，她通过区别ie不同的版本给html标签添加识别class，然后针对兼容问题进行样式覆盖。

不同的浏览器、同浏览器不同版本对css的支持都是不一样的，特别是在使用一些css3样式一定要了解它的支持情况，不然很容易造成不同浏览器的兼容问题。例如伪类：read-only (选择器用于选取设置了 “readonly” 属性的元素)IE是完全不支持它如果你把样式写成如下：

.test,
.test:read-only{
  color:red;
}

好了，在IE下就直接玩完了test的样式就直接没有了。所有像上面多个class写一起的时候一定要注意其中是否有完全不支持的样式
